Συμπεριλαμβάνεται makefile για τη μεταγλώττιση.
Εκτέλεση: ./bet για login ή ./bet -R για register

Όσον αφορά τους τύπους κατηγοριών , δεν ακολουθήθηκε η κληρονομικότητα μεταξύ των κυρίως κλάσεων, 
αλλά υλοποιήθηκαν ως friend. Ωστόσο χρησιμοποιείται μια υπερκλάση Location από την οποία
κληρονομούν όλες οι κατηγορίες και λειτουργεί ως μέσο αναφοράς για όλες.

Όσον αφορά τους τύπους χρηστών, ακολουθήθηκε η κληρονομικότητα ως εξής:
  User)	->Guest
	->RegUser)->Punter
		  ->Trader
		  ->Director

Τέλος χρησιμοποιείται ένα struct DataNode με δείκτες στα διάφορα διαχειριζόμενα δεδομένα
λειτουργώντας ως "βιβλιοθήκη" για την κατά το δυνατό αποφυγή ορισμού global μεταβλητών και
υπερπληθόρας ορισμάτων συναρτήσεων.

Οι invisible κόμβοι χαρακτηρίζονται στο Hierarchy.dat με ένα 'v' στην αρχή της σειράς π.χ:
1.2 Greece - Greek Super League
V1.2.1 Kalloni v PAOK
--------------------------------------------------------------------------------------------------
Αρχικά ξεκινήσαμε υλοποιώντας βασικές λειτουργίες ο ένας περί categories και ο άλλος περί users, ακολούθησαν τεστ για τη λειτουργία των βασικών αντικειμένων, έπειτα login, register και σύνδεση όλων
σε ένα ενιαίο project το οποίο διαχειριζόμασταν απο κοινού και μοιραζόμασταν-ενημερώναμε μέσω cloud για κάθε περαιτέρω προσθήκη,υλοποίηση ή βελτιστοποίηση μέχρι τέλους.

Υλοποίθηκε σε DevC++ Windows και ενδέχεται περιστασιακά να παρουσιαστούν μικρο-bugs κατά τη λειτουργία σε linux.
